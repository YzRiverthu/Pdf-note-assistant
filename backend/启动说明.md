# 本地 OCR 后端启动说明

## 运行前准备

建议先在项目根目录完成依赖安装：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## 单独启动后端

在项目根目录执行：

```bash
python -m uvicorn backend.paddle_ocr_server:app --host 127.0.0.1 --port 8765
```

## 一键启动前后端

如果希望前端和后端一起启动，在项目根目录执行：

```bash
bash start_app.sh
```

说明：

- `start_app.sh` 会优先使用 `python3`，找不到时回退到 `python`
- 脚本会同时启动 OCR 后端和前端静态服务

启动后：

- 前端地址：`http://127.0.0.1:8080/`
- OCR 接口：`http://127.0.0.1:8765/ocr`

## 健康检查

浏览器或命令行访问：

```bash
curl http://127.0.0.1:8765/health
```

## 前端默认地址

前端 `OCR 服务地址` 默认就是：

```text
http://127.0.0.1:8765/ocr
```

## 说明

- 该服务会把 PaddleX 缓存和 matplotlib 缓存写到 `backend/.runtime/`
- 前端页面现在建议通过 `http.server` 提供，而不是直接双击本地文件
- 第一次真正调用 OCR 时，PaddleOCR 可能会下载模型文件
- 如果本地网络受限，首次模型下载可能失败
- `paddlepaddle` 是安装包名，运行时代码实际导入模块名是 `paddle`
