# PDF 标注工具教学文档

## 1. 工具用途

这个工具用于对 PDF 论文进行人工标注，并将标注结果实时组织成 JSON Schema 对应的结构化数据。

它适合做这类工作：

- 打开论文 PDF
- 在页面上框选证据区域
- 选择该区域对应的字段标签
- 提取或手工修正字段值
- 保存标注
- 实时查看当前 JSON 结果
- 最后导出 JSON

## 2. 页面结构

页面主要分为 3 个区域：

1. 顶部工具栏
用于加载 PDF、上传标注 Schema、上传辅助注释 Schema、重置 Schema、翻页、缩放、截图、切换 JSON 视图、查看辅助注释、校验和导出，也可配合本地 OCR 后端在框选后自动识别文本。

推荐启动方式：

- 前后端一起启动：`bash start_app.sh`
- 然后在浏览器打开：`http://127.0.0.1:8080/`

2. 左侧主视图
默认是 PDF 视图，用来浏览文档和框选区域。
也可以切换为 JSON 预览视图，用来看当前结构化结果。

3. 右侧操作面板
用于选择字段标签、填写字段值、查看校验结果和管理标注列表。

## 3. 顶部工具栏说明

### 3.1 选择 PDF

点击文件选择框，上传本地 PDF 文件。

上传后会自动：

- 渲染第一页
- 显示页码
- 清空上一份文档的标注状态
- 初始化 JSON 结果

### 3.2 上传标注 Schema / 恢复默认 Schema

你可以在顶部工具栏上传自己的 `json_schema.json` 文件，工具会自动解析字段层级并重建右侧标签树。

适用场景：

- 不同项目使用不同字段结构
- 需要临时切换到另一套标注模板
- 想直接拿自己维护的 schema 做人工标注

说明：

- 标注 Schema 上传入口主要面向普通 `JSON`
- 如果上传的是标准 `JSON Schema`，会优先根据 `properties` / `items` 生成字段路径
- 如果上传的是“模板对象”形式的 schema，也可以直接解析
- 切换 Schema 时，当前草稿和已保存标注会被清空，避免路径错位
- 点击“恢复默认 Schema”可回到工具内置字段树

### 3.3 上传辅助注释 Schema

你还可以额外上传一个 `json_schema.jsonc` 文件。

这个文件不会参与字段树生成，而是作为“辅助阅读材料”保存下来，供标注时随时查看。

适用场景：

- 字段命名较抽象，不容易直接理解
- 需要查看字段旁边的中文解释或备注
- 想一边标注，一边参考带注释的原始 schema

上传后：

- 顶部会显示当前已加载的辅助注释文件名
- “查看辅助注释”按钮会变为可用
- 可以在 PDF、JSON 结果、辅助注释三个视图之间来回切换

### 3.4 上一页 / 下一页

用于切换 PDF 页码。

### 3.5 缩小 / 放大

用于调整 PDF 显示比例。

说明：

- 放大后可更精细地框选
- 工具已经做了高清渲染，放大后清晰度会更高

### 3.6 截图

点击“截图”后，工具会进入截图模式。

使用方式：

1. 点击顶部“截图”按钮
2. 在当前 PDF 页面上拖拽框选一个区域
3. 松开鼠标后，工具会立即把这个区域下载为 PNG 图片

截图特点：

- 截取的是当前页中被框选的区域
- 会同时包含 PDF 内容和该区域内已显示的标注框
- 仅在 PDF 视图下可用

### 3.7 查看 JSON / 返回 PDF

点击后可切换两个视图：

- `PDF` 视图：用于框选和标注
- `JSON` 视图：用于实时查看当前结构化结果

JSON 视图展示的是当前已保存标注生成的最终结构化结果，不需要先导出文件。

### 3.8 查看辅助注释 / 返回 PDF

点击后可以打开辅助注释视图。

这个视图会完整显示你上传的 `json_schema.jsonc` 内容，包括注释文本，便于边看边理解字段含义。

说明：

- 未上传辅助注释文件前，按钮不可用
- 切换到辅助注释视图不会清空当前草稿
- 看完后可直接返回 PDF 继续标注

### 3.9 校验

点击后会对当前标注结果做基础检查。

当前主要检查：

- 是否缺少字段路径
- 是否存在数组字段但未填写索引
- 是否存在空值
- 同一路径是否被标注为不同值

作用是帮助你在导出前发现明显问题。

### 3.10 导出 JSON

点击后导出当前标注结果为 JSON 文件。

导出的内容就是“查看 JSON”中当前展示的最终结果本身。

## 4. 基本标注流程

推荐使用以下顺序：

1. 上传 PDF
2. 选择字段标签
3. 二选一：
   - 在 PDF 中框选对应区域，自动提取 OCR 文本
   - 不框选，直接手动填写内容
4. 检查或填写原文与字段值
5. 点击“保存标注”
6. 重复以上过程
7. 切换到 JSON 视图检查结果
8. 校验并导出

说明：

- 也可以先框选，再选标签
- 也可以完全不框选，直接手动输入后保存
- 但保存前必须选定最终字段

如果只是保存图片证据，也可以使用另一条流程：

1. 上传 PDF
2. 点击“截图”
3. 在页面上框选一个区域
4. 自动下载截图

## 5. 内容分类区域说明

右侧“内容分类”区域用于选择目标字段。

### 5.1 多级标签

字段不是只有一级、二级，而是按实际 schema 层级逐级展开。

例如可能出现：

- 第 1 级：`metadata`
- 第 2 级：`alloy_system`
- 第 3 级：`nominal_composition`
- 第 4 级：`weight_percent`

你需要一直选择到最终字段为止。

### 5.2 进度提示

每一级选项后面会带提示，例如：

- `3/12`：表示该分组下总共有 12 个末级字段，已经标了 3 个
- `●`：该末级字段已经标过
- `○`：该末级字段还没标过

这样可以帮助你快速判断哪些字段已经完成，哪些还没做。

### 5.3 数组索引

只有当当前选中的字段属于数组项时，才会显示“数组索引”输入框。

例如这类字段可能需要索引：

- `authors[]`
- `synthesis_methods[]`
- `heat_treatment[]`

如果路径中有多层数组，系统会为每一层分别生成一个索引输入框。

例如：

- `experiments[].samples[].name`

这时你需要分别填写：

- `experiments[]` 的索引
- `experiments[].samples[]` 的索引

如果当前字段不是数组字段，这一项不会显示。

### 5.4 最终字段路径

“最终字段路径”是系统根据你当前选择自动生成的只读结果。

作用：

- 告诉你当前标注将写入 JSON 的哪个字段
- 便于排查路径是否选对

用户不需要手工输入路径。

### 5.5 自定义字段

每一级标签右边都有一个 `+` 按钮。

作用：

- 在当前层级下新增一个自定义字段
- 输入字段名后，该字段会立刻加入当前标签选项
- 可以像普通字段一样继续标注它的 value

适用场景：

- schema 中暂时没有这个字段
- 论文里出现了需要补充记录的新字段

限制：

- 字段名不能包含 `.`
- 字段名不能包含 `[` 或 `]`

## 6. 提取值区域说明

### 6.0 OCR 服务地址

“OCR 服务地址”用于填写本地 OCR 后端接口地址。

当前默认值：

- `http://127.0.0.1:8765/ocr`

如果你已经启动了本地 OCR 服务，通常不需要修改。

后端启动方式见：

- [启动说明.md](/Users/mac/Desktop/paper_extractor/标注工具/backend/启动说明.md)

### 6.1 原文摘录

这是系统从你框选区域中自动提取的文本。

你可以：

- 直接使用自动提取结果
- 手工修正提取错误
- 手工填写文本

### 6.2 字段值

这是最终写入 JSON 的值。

当前版本中：

- 字段值统一按 `string` 保存
- 不强制自动转 number / boolean

如果原文摘录本身就适合作为结果，可以直接复制或沿用。

### 6.3 置信度

用于记录你对当前标注结果的可信程度。

输入范围：

- `0` 到 `1`

例如：

- `1` 表示非常确定
- `0.6` 表示不太确定

### 6.4 备注

用于记录补充说明，例如：

- 文本提取不完整
- 需要复核
- 该值是人工整理后的表达

### 6.5 重新提取文本

点击后会再次对当前框选区域执行文本提取。

适用情况：

- 你修改了框选区域
- 你想重新尝试提取

### 6.6 自动 OCR 识别

当你在 PDF 中完成框选后，工具会自动：

- 直接把当前框选区域发送到本地 OCR 服务识别
- 将 OCR 结果回填到“原文摘录”

适用情况：

- PDF 文本层提取不到内容
- 提取结果明显不完整
- 当前 PDF 更像扫描件或图片型 PDF

识别成功后：

- 结果会回填到“原文摘录”
- 如果“字段值”还是空的，也会自动回填到“字段值”

如果本地 OCR 服务没有启动，工具会保留本地文本提取结果，不会阻塞后续标注。

### 6.7 清空当前框选

点击后会清除当前草稿框选和提取内容，但不会清除当前已经选好的字段标签。

### 6.8 保存标注

点击后会把当前字段路径、原文、字段值等信息保存为一条正式标注。

保存后会：

- 出现在“标注列表”中
- 自动进入实时 JSON 结果

补充说明：

- 如果当前有框选区域，会一起保存该证据框
- 如果当前没有框选，也可以直接手动填写并保存

### 6.9 取消编辑

用于退出当前编辑状态。

说明：

- 不会删除已保存标注
- 一般用于放弃当前草稿或退出已有标注的编辑

## 7. 框选功能说明

在 PDF 页面上，用鼠标拖拽即可框选区域。

框选后系统会：

- 记录该区域的位置
- 自动提取区域内文本
- 在页面上高亮显示该框

当前框选支持：

- 先选标签，再框选
- 先框选，再选标签
- 点击“截图”后进入截图模式，再框选截图区域

但保存前必须满足：

- 已有框选区域
- 已选定最终字段

说明：

- 截图模式下，框选不会生成标注草稿
- 截图会在框选完成后直接下载

## 8. 标注列表说明

每保存一条标注，都会出现在右侧“标注列表”中。

每条标注会显示：

- 对应的字段层级
- 页码
- 标注 ID
- 最终字段路径
- 原文或字段值摘要

每条记录支持两个操作：

### 8.1 编辑

点击后会：

- 跳转到对应页
- 高亮原标注框
- 回填当前字段、文本和值

方便修改已有标注。

### 8.2 删除

点击后删除该条标注，并同步从 JSON 结果中移除。

## 9. JSON 实时预览说明

点击顶部“查看 JSON”后，会进入 JSON 预览页。

这个页面的作用是实时展示当前结构化结果。

特点：

- 展示的是当前 `data`
- 保存或删除标注后会自动更新
- 适合随时检查字段是否落到了正确位置

你可以用它来确认：

- 路径是否正确
- 数组索引是否正确
- 值是否写到了预期字段下

## 10. 导出的 JSON 包含什么

导出文件中主要包含两部分：

### 10.1 `data`

表示最终结构化结果。

这是按 schema 路径写入后的 JSON 数据。

### 10.2 `annotations`

表示全部证据级标注记录。

其中包含：

- 页码
- 框坐标
- 字段路径
- 原文摘录
- 字段值
- 备注
- 置信度

这部分用于追溯和复查。

## 11. 常见使用建议

### 11.1 先标简单字段

建议先从这些字段开始：

- 标题
- DOI
- 期刊
- 发表年份

这样可以先确认字段路径理解是正确的。

### 11.2 遇到数组字段时及时填写索引

例如作者、热处理步骤、相组成这类字段，通常不是单个值。

如果系统出现“数组索引”，要及时填写正确的编号。

### 11.3 框选不准时先放大

如果当前页面文字太密或太小：

- 先点击“放大”
- 再进行框选

这样更容易提取准确文本。

### 11.4 用 JSON 视图做中途检查

标注到一半时，不要等到最后再导出检查。

建议随时切换到 JSON 视图看一眼，确认：

- 值是否落位
- 层级是否正确
- 数组是否写到了预期位置

## 12. 当前版本已知特点

当前版本是一个最小可用版本，特点是：

- 支持人工框选
- 支持多级字段选择
- 支持自定义字段
- 支持数组索引
- 支持实时 JSON 预览
- 支持截图模式
- 支持基础校验和导出

当前版本暂不强调：

- OCR
- 自动字段抽取
- 多人协作
- 复杂校验规则

## 13. 一句话总结

这个工具的核心用法可以概括为：

选择字段 -> 框选证据 -> 修正文本和值 -> 保存标注 -> 实时查看 JSON -> 导出结果
